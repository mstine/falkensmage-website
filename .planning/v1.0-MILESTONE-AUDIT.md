### Unsatisfied / Partial Requirements Detail

#### PERF-03: Font preload with crossorigin (Phase 3) — PARTIAL

Preload links exist in `baseof.html` pointing to `/fonts/cinzel-latin-wght-normal.woff2` and `/fonts/space-grotesk-latin-wght-normal.woff2`. However, Hugo's `css.Build` pipeline copies `assets/fonts/` files into `public/css/` with fingerprinted filenames and rewrites the `@font-face` src to relative fingerprinted paths. The preload href and `@font-face` src resolve to different URLs — the browser preload cache misses, and fonts are downloaded twice.

**Impact:** Preload hint is wasted. Fonts load via the CSS-triggered path only. No FOIT (font-display: swap handles it) but the intended performance benefit of preloading is not realized.

#### PERF-01: Sub-1s page load on 3G (Phase 3) — PARTIAL

Dependent on PERF-03. The font double-fetch adds unmeasured latency. No Lighthouse/WebPageTest measurement has been run. The structural prerequisites are correct (no external CDN, no analytics, no JS) but the actual 3G timing is unconfirmed.

#### A11Y-01: Semantic HTML5 landmarks (Phase 3) — PARTIAL

`index.html` wraps the footer partial call in `<footer>`. The partial `footer.html` itself uses `<footer>` as its root element. Built HTML contains nested `<footer>` inside `<footer>` — invalid per HTML5 spec. Screen readers may misparse the landmark tree.

### Font Preload Mismatch (Cross-Phase)

Phase 1 created fonts in `assets/fonts/` (for css.Build processing) and `static/fonts/` (for direct serving). Phase 3 added preload links pointing to `/fonts/` (static path). Hugo's `css.Build` rewrites `@font-face` src to fingerprinted paths under `/css/`. The two paths diverge — preload is wasted.
