## Overview

## Phases

### Phase 1: Theme Foundation

**Goal:** The `arcaeon` Hugo theme exists as a portable design system ‚Äî color tokens, typography, glow treatments, dark graphic novel aesthetic, and responsive scaffold ‚Äî ready for sections to be dropped into.
**Depends on:** Nothing (first phase)
**Requirements:** THEME-01, THEME-02, THEME-03, THEME-04, THEME-05, THEME-06, THEME-07, INFRA-01, INFRA-02
**Plans:** 3 plans

Plans:

- [x] 01-01: Hugo project init + ARCAEON CSS token system
- [x] 01-02: Typography + Glow treatment system
- [x] 01-03: Dark aesthetic, sigils, kitchen sink demo + visual checkpoint

**Details:**

Hugo Extended 0.160.1 project initialized with portable theme under `themes/arcaeon/`. ARCAEON palette materialized as CSS custom properties with two-tier naming: palette truth (`--arcaeon-*`) plus semantic aliases (`--color-*`). Triad Rule enforced via section-scoped CSS variable classes (`.section-void`, `.section-depth`). Self-hosted Cinzel Variable (display, 400‚Äì900) and Space Grotesk Variable (body, 300‚Äì700) WOFF2 fonts via `@fontsource-variable`, declared with `font-display: swap` for FOIT prevention. Three GPU-composited glow variants (ambient pulse, interactive hover, Radiant Core CTA gradient) using the `::before` + `opacity` pattern ‚Äî no `box-shadow`. Pseudo-element convention locked: `::before` = glow, `::after` = sigil. Minor Third type scale (1.2 ratio) with `clamp()` display treatment. Every animation wrapped in `prefers-reduced-motion: reduce`. Draft-only kitchen sink demo page rendering every design system element for 375px visual validation.
