## Spacing Scale

Source: Established in Phase 1. The existing `.section` and `.section-content` classes define all live spacing. Phase 3 adds only the "Currently" card — apply the same scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing (Currently card inner gaps) |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section-level gaps, card body spacing |
| xl | 32px | Section padding base (`.section` 3rem → 48px on mobile) |
| 2xl | 48px | Section vertical padding on mobile (`.section`: `3rem`) |
| 3xl | 64px | Section vertical padding on desktop (`.section`: `5rem`) |

**Currently card padding:** `1rem` (16px) horizontal, `1rem` (16px) vertical — aligns with the `md` token. Matches `.social-card` padding convention.

**Currently card separator gap (between post link and focus blurb):** `1rem` (16px) — `md` token, consistent with default element spacing.

**Exceptions:** Font preload links, OG meta tags, semantic landmark restructure, and image srcset are structural/head changes — no spacing tokens apply.

## Typography

### Phase 3 Normative Contract

The following sizes and weights are directly used by Phase 3 components (the Currently section). This is the normative contract for this phase.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Label / caption | 14px (0.875rem) | 400 | 1.5 | Space Grotesk Variable |
| Body / link text | 16px (1rem) | 400 or 600 | 1.6 | Space Grotesk Variable |
| Section heading (h3) | 23px (1.44rem) — `var(--text-h3)` | 600 | 1.2 | Cinzel Variable |

**Phase 3 weight set:** 400 (regular) and 600 (semibold). Weight 600 applies to both the "CURRENTLY" section heading in Cinzel and the post title link / fallback link in Space Grotesk. Both fonts are variable — 600 renders correctly in each.

**Currently section typography specifics:**
- Section header label ("CURRENTLY"): `var(--text-h3)` (23px) at weight 600, Cinzel Variable — matches established h3 semibold pattern
- "Latest from Feral Architecture:" sub-label: 14px, weight 400, Space Grotesk, opacity 0.7
- Post title link: 16px, weight 600, Space Grotesk, color `var(--color-text-accent)` (Ion Glow `#5be7ff`)
- Guillemets treatment: `«` and `»` rendered as Unicode characters flanking the post title — same font/size as post title
- Current focus blurb: 16px, weight 400, Space Grotesk, opacity 0.85
- Fallback link ("Read Feral Architecture →"): 16px, weight 600, color `var(--color-text-accent)`

### Phase 1 Inherited Reference (established — not new in Phase 3)

The following sizes exist in the codebase from Phase 1. They are documented here for completeness only. The Phase 3 contract does not introduce or extend these.

| Role | Size | Weight | Font |

## Currently Section — Visual Contract

Source: D-01 through D-05 (locked decisions from CONTEXT.md), specifics section.

**Section assignment:** `.section-depth` (Deep Indigo background). Not full-width — treated as a contained information module inside standard `.section` + `.section-content` layout.

**Card treatment:** Single card block sitting inside `.section-content`. Applies the same ghost card visual language as social cards:
- Border: `1px solid rgba(122, 44, 255, 0.35)` at rest
- Border on hover: `1px solid #7a2cff`
- Border radius: `8px` (matches social card)
- Background: none (ghost — section background shows through)
- Padding: `1rem 1rem` (16px vertical, 16px horizontal) — `md` token on all sides
- No `.glow-interactive` on the card wrapper itself — glow is reserved for the post title link inside it

**Content layout (top to bottom inside card):**
1. Section label: "CURRENTLY" — Cinzel h3, weight 600, `var(--color-text)`, margin-bottom `0.5rem` (8px)
2. Sub-label: "Latest from Feral Architecture:" — Space Grotesk 14px, weight 400, opacity 0.7, margin-bottom `0.25rem` (4px)
3. Post title: `« {title} »` — Space Grotesk 16px, weight 600, color Ion Glow, `.glow-interactive` on the `<a>` tag, opens in new tab
4. Separator: `1rem` (16px) gap — `md` token
5. Current focus blurb: Space Grotesk 16px, weight 400, opacity 0.85, sourced from `_index.md` front matter `currently_focus` param

**Fallback state (RSS unreachable):**
- Replaces steps 2–3 with: "Read Feral Architecture →" — Space Grotesk 16px, weight 600, Ion Glow color, links to `https://feralarchitecture.substack.com`
- Current focus blurb still renders (not RSS-dependent)
- No error UI shown to end user — fallback is seamless

**Landmark placement (D-09 decision):**
```html
<header>  ← wraps Hero section containing <h1>
<main>    ← wraps Currently + Identity+CTA + Social sections
<footer>  ← wraps Footer section
```
No `<nav>` — single-page, no navigation links.

**index.html partial order after Phase 3:**
```
{{ partial "sections/hero.html" . }}       ← inside <header>
{{ partial "sections/currently.html" . }}  ← first inside <main>
{{ partial "sections/identity-cta.html" . }}
{{ partial "sections/social.html" . }}
{{ partial "sections/footer.html" . }}     ← inside <footer>
```

## Performance Contract

Source: PERF-01 through PERF-04 requirements + D-06 through D-08 decisions.

**Hero image pipeline (D-06):**
- Source: `~/Documents/Business-Brand/Feral-Architecture/covers/2026-04-14-aesthetic-decisions/final/aesthetic-decisions-cover.jpg` (1424x752)
- Mobile srcset: `.Fill "750x580 Center"` → `.Process "webp"` + JPEG fallback
- Desktop srcset: `.Resize "1200x q85"` → `.Process "webp"` + JPEG fallback
- OG image: `.Fill "1200x630 Center"` → `.Process "webp"` (or JPEG for OG — OG consumers vary on WebP support; provide JPEG)
- Element: `<picture>` with `<source>` for WebP + JPEG at each breakpoint
- Loading: `loading="eager"` (above fold, no lazy-load)

**No other below-fold images exist** (D-08 decision) — social icons are inline SVG, no `loading="lazy"` needed anywhere else.

**Font preloading (D-07):**
Add to `<head>` in `baseof.html`:
```html
<link rel="preload" href="/fonts/cinzel-latin-wght-normal.woff2"
      as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/fonts/space-grotesk-latin-wght-normal.woff2"
      as="font" type="font/woff2" crossorigin>
```
Note: Hugo `css.Build` resolves font URLs in CSS. Preload hints must use the static path directly (not the fingerprinted CSS URL). Font files live at `themes/arcaeon/static/fonts/` → served as `/fonts/`.

**External dependency count target: 0** — No external CDN calls, no analytics, no external JS. Build-time RSS fetch via `resources.GetRemote` (no runtime network call).

Already implemented in Phase 2 hero partial. Verify it persists through srcset restructure.

**Contrast audit targets (D-11, A11Y-02):**
| Text | Background | Token | Minimum Ratio |
|------|------------|-------|---------------|
| Solar White `#fff4e6` | Void Purple `#1a0f2e` | body text | AA (4.5:1) |
| Solar White `#fff4e6` | Deep Indigo `#0a0f3c` | body text | AA (4.5:1) |
| Ion Glow `#5be7ff` | Void Purple `#1a0f2e` | link text | AA (4.5:1) |
| Ion Glow `#5be7ff` | Deep Indigo `#0a0f3c` | link text | AA (4.5:1) |
| Electric Violet `#7a2cff` | any | NEVER body text | decoration only |
| Neon Magenta `#ff3cac` | any | NEVER body text | decoration only |

**Motion suppression (D-12, A11Y-04):**
`prefers-reduced-motion` media query already wired in Phase 1 for:
- `.glow-ambient::before` — animation: none
- `.glow-interactive::before` — transition: none
- `.glow-radiant-core::before` — transition: none
- `.social-card` — transition: none
- `html { scroll-behavior: auto }` — smooth scroll disabled

Verify Phase 3 adds no new animations. The Currently card hover (border-color transition) must also respect `prefers-reduced-motion`:
```css
@media (prefers-reduced-motion: reduce) {
  .currently-card {
    transition: none;
  }
}
```

**Focus visible:** All interactive elements must show `:focus-visible` state. The existing `.glow-interactive:focus-visible::before` pattern handles social cards and CTA. Apply same pattern to the Currently post title link.
