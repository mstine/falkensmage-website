{{/* Currently Section — RSS + current focus (CURR-01, CURR-02, CURR-03) */}}
{{ $rssURL := "https://feralarchitecture.com/feed" }}
{{ $latestPost := dict }}

      {{ if $latestPost.title }}
        <p class="currently-sublabel">Latest from Feral Architecture:</p>
        <a href="{{ $latestPost.link }}"
           class="currently-post-link glow-interactive"
           target="_blank"
           rel="noopener noreferrer"
           aria-label="Read: {{ $latestPost.title }} on Feral Architecture">

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build-time RSS fetch | External data (feralarchitecture.com/feed) enters Hugo build pipeline via `resources.GetRemote` |
| RSS title rendering | RSS item `.title` value rendered into HTML output |
| RSS link rendering | RSS item `.link` value rendered as `href` attribute |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | RSS feed URL | accept | Feed URL is hardcoded to `feralarchitecture.com/feed` (Matt's own domain); no user input controls the URL |
| T-03-02 | Tampering | RSS title content | mitigate | Hugo template engine HTML-escapes all output by default; do NOT use `safeHTML` on RSS title — let default escaping handle XSS. `htmlUnescape` only decodes entity references, does not bypass escaping |
| T-03-03 | Tampering | RSS link content | mitigate | Hugo template engine escapes attribute values in `href` by default; do NOT use `safeURL` on RSS link value |
| T-03-04 | Information Disclosure | Build-time errors | mitigate | `warnf` logs RSS errors to build output only, not to rendered HTML; no error UI shown to end users |
| T-03-05 | Denial of Service | RSS fetch timeout | mitigate | Hugo `resources.GetRemote` has default timeout; `try` wrapping prevents build failure; fallback content renders if fetch fails |
| T-03-06 | Elevation of Privilege | n/a | accept | Static HTML site, no server-side execution, no user sessions, no input processing |
</threat_model>
